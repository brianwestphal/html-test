Attribute selectors — quoting, case flags, escapes

Exact + case flags

data-state="active" → green bg + outline
data-state="Active" → outline only (case-insensitive [i])
data-state="ACTIVE" → outline only
data-state="inactive" → no match

Case-sensitive [s] flag

data-mode="dark" — matches
data-mode="DARK" — does NOT match (s)
data-mode="Dark" — does NOT match (s)

Substring matching: *= / ^= / $=

btn primary (bold via *= "primary")
primary-cta (bold)
btn-large (left bar via ^="btn-")
btn-lg (left bar AND large via $="-lg")
card-lg (large only)

Word match [~=] vs partial [*=]

classes "info warn alert" — yellow bg ([~="warn"])
"warning" — does NOT match ~=warn (whole word)
"warn-x warned" — no whole-word match

Hyphen prefix [|=] for language

lang="en" — italic
lang="en-US" — italic
lang="en-GB-x-foo" — italic
lang="enchanted" — NOT italic
lang="fr" — NOT italic

Escapes inside quoted values

data-q='he said "hi"' — red text via escaped quotes
no match
