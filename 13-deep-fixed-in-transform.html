position:fixed inside a transformed / filtered ancestor

When an ancestor establishes a containing block for fixed-positioned descendants (via transform, filter, perspective, will-change: transform, or contain: paint), the descendant becomes effectively absolutely-positioned to that ancestor instead of to the viewport.

Reference — fixed pinned to viewport

Plain ancestor — no transform/filter/etc.

FIXED → viewport

transform breaks fixed → CB is the .frame

ancestor: transform: translate(0,0)

FIXED → trapped

filter breaks fixed

ancestor: filter: blur(0)

FIXED → trapped

perspective breaks fixed

ancestor: perspective: 600px

FIXED → trapped

will-change: transform breaks fixed

ancestor: will-change: transform

FIXED → trapped

contain: paint breaks fixed

ancestor: contain: paint

FIXED → trapped

Pass: in the reference, the red box pins to the bottom-right of the viewport (or section, when scrolling). In every other case, the red box sits at the bottom-right of the dashed yellow .frame instead.
