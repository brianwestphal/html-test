z-index on flex / grid items (no position)

Flex and grid items can use z-index directly — they don't need position: relative. Each item with a non-auto z-index becomes a stacking context.

Flex — overlapping via negative margin, default tree-order paint

A (default)
B (default)
C (default)
D (default)

Without z-index, paint order = source order: D on top.

Flex — z-index reorders

A z:4 (front)
B z:1
C z:3
D z:2

Grid — z-index also works without position

A z:2
B z:4 (front)
C z:1
D z:3

order rearranges layout but NOT paint without z-index

order moves items in the flex flow. Paint order, however, stays in source order until z-index is set. Look: visual flow is D, C, B, A but paint stack still goes A → D, so D paints in front of overlap.

A (order 4)
B (order 3)
C (order 2)
D (order 1, painted last)

NEGATIVE — z-index on a non-flex/grid in-flow block does nothing

Plain block siblings, no positioning, no flex/grid parent. z-index here is ignored; paint = source order: B is in front of A.

A z:99 (ignored)
B z:1 (still on top)
